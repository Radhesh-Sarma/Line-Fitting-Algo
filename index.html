We have successfully implemented the Segmented Least Squares Algorithm in C and visualised our results using Python. The gist of the algorithm is to fit the optimal number of segments to a given set of data points while taking into account the cost of each added segment and the least square error generated by the fit. This is done using a recursive algorithm which computes the optimal segmentation of points one by one and then compiles all that information to output the segments which fit the data with the least combined error.

Analysis

The Running Time and Memory Usage both increase with increasing number of points
When comparing the cases with n=10 and n=1000, we observe that the running time increases for the larger test sample, this is because a larger number of points implies that the code will have to perform more computations. Similarly, the memory usage also increases with increasing n, this is because the size of the error matrix to be computed increases as n^2.

This aligns with our understanding that the code will have to compute the error for all the points irrespective of whatever penalty value we provide. On the other hand, the memory usage decrease with increasing penalty value. We speculate that this occurs because the number of segments in the optimal solution decreases drastically with increasing penalty value. Therefore, a smaller number of segments have to be stored for a larger penalty value, resulting in a smaller memory usage.
